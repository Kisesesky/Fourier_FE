# 2026-02-02 작업 요약

## 변경 사항
- 캘린더 사이드바 패널을 문서 트리 스타일로 재구성하고, 폴더/캘린더 우클릭 메뉴 및 폴더 추가 모달을 다크모드 대응으로 정비.
- 캘린더 항목 드래그&드롭으로 폴더 이동을 지원하고, 배지(다가오는 일정 수) 표시 로직과 스타일을 조정.
- 캘린더 생성/관리 흐름 정리: 개인 캘린더 자동 생성/삭제 제한, 개인 캘린더 UI 편집/삭제 비활성, 프로젝트 생성 시 기본 폴더/캘린더 명칭 변경.
- 캘린더 헤더의 카테고리 UI를 칩 형태로 변경하고, 모달 기반 카테고리 추가/색상 선택 UI를 추가.
- 전체보기에서 카테고리/이벤트 로딩을 분리해 안정적으로 로드되도록 개선하고, 프로젝트 전체 카테고리 조회 API 추가.

## 백엔드 변경
- 프로젝트 전체 카테고리 조회 API(`/projects/:id/calendar/categories`) 추가 및 개인 캘린더 제외 로직 반영.
- 개인 캘린더 자동 생성/삭제 제한, 프로젝트 멤버 추가/삭제 시 개인 캘린더/이벤트 처리 로직 강화.
- 캘린더 폴더 기본 이름/프로젝트 캘린더 기본 이름 정책 변경.

## 프론트엔드 변경
- 사이드바 캘린더 트리 UI 스타일/동작 개선(색상, 크기, 배지, 다크모드).
- 전체보기 카테고리 로딩 안정화 및 이벤트 로딩 분리.
- 카테고리 추가 모달 도입 및 색상 선택 UX 통일.

## 남은 작업/개선 사항
- `/calendar/categories` 호출이 실제로 3001 API로 나가는지 네트워크 탭에서 확인할 수 있도록 모니터링 가이드 추가.
- 카테고리 리스트에서 동일 이름+색상 중복 표시 정책을 서버/클라이언트 중 어디에서 처리할지 확정 후 정리.
- 전체보기 카테고리 칩 UI/정렬 규칙과 빈 상태(카테고리 없음) 디자인 확정.
